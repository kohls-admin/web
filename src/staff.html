üî® Moderation Actions & When to Use Them

These actions should be used progressively, starting with the least severe and escalating as needed. Always prioritize clear communication and warnings before taking action, unless the situation demands immediate intervention.

Timeout:
  • Purpose: A temporary mute to give a user a chance to cool down.
  • When to Use:
    • Minor disruptive behavior (spamming, getting heated).
    • When a user needs a short break.
    • As a first step before stronger actions.
  • Duration: Start short (5-10 minutes), increase for repeat offenses.
Kick:
  • Purpose: Removes a user, they can rejoin, but it's a strong warning.
  • When to Use:
    • Repeated minor offenses after timeouts.
    • Inappropriate behavior not warranting a ban.
    • Compromised accounts doing harm (e.g., phishing).
Softban:
  • Purpose: Kicks and deletes some of their recent messages.
  • When to Use:
    • Lots of disruptive content needs removal (spam, slurs, etc.).
    • Repeated offenses worse than a kick, but not ban-worthy.
Ban:
  • Purpose: Permanently removes a user.
  • When to Use:
    • Severe or repeated rule violations (hate speech, harassment).
    • User clearly won't follow the rules.
    • Extreme disruption harming the community.

üîÄ Handling Issues & Feature Requests

Help us keep things organized by moving issues and feature requests from Discord to GitHub.

  • When to Move:
    • If a user reports a bug or suggests a new feature in the #bugs or #feedback channels, guide them to create a GitHub issue.
  • Why it Matters:
    • Centralizes bug reports and feature requests for better tracking and management.
    • Allows developers to directly address issues and consider new ideas.
    • Keeps Discord discussions focused on general support and community interaction.
  • How to Help:
    • Politely inform the user that their feedback is valuable and should be reported on GitHub.
    • Offer assistance if they need help navigating GitHub or creating an issue.
    • If necessary, create the GitHub issue on their behalf, ensuring you accurately capture their feedback.

By streamlining this process, we can ensure that all valuable feedback reaches the right place and contributes to the ongoing improvement of Kohl's Admin.
